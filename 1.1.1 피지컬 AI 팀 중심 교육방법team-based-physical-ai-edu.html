Deep Dive

교육 타입 5가지 상세

각 타입은 서로를 보완하며 현장형 역량을 키웁니다.

01
🎯
팀 프로젝트 (Team Project)
팀이 하나의 현장 문제를 끝까지 해결하는 핵심 교육 방식
CORE

팀이 하나의 현장 문제를 선정하고, 데이터 수집 → AI 모델 학습 → 로봇 적용까지 전 과정을 직접 수행합니다. 이 교육의 가장 큰 핵심이며, 다른 모든 활동은 이 프로젝트에 수렴합니다.

현장 적용 예시
🦾
로봇팔(SO-ARM101): 컬러/위치 기반 물체 분류 후 자동 집기
🚗
자율주행 로봇(LeKiwi): 장애물 회피 + 목적지 자동 이동
📷
RealSense 카메라: 깊이(Depth) 기반 물체 거리 인식 후 안전 정지
👉 결과: 실제 동작하는 AI 시스템 구현
02
🧠
팀 미션 + 토의 (Team Mission & Discussion)
팀이 함께 해결 전략을 토의하고 설계 방향을 결정
DESIGN

주어진 문제에 대해 팀이 해결 전략을 토의하고, 설계 방향을 결정하는 과정입니다. 어떤 접근이 옳은지 정답이 없는 문제를 다루며, 토론을 통해 최적해를 찾는 훈련을 합니다.

토의 주제 예시
🦾
로봇팔: "잡기 실패 원인은 데이터 부족인가? 모델 문제인가?"
🚗
자율주행: "카메라 vs LiDAR — 어떤 센서를 선택할 것인가?"
📷
RealSense: "2D 이미지보다 Depth를 써야 하는 이유는 무엇인가?"
👉 결과: AI 시스템 설계 능력 향상
03
🔬
개인 미션 + 토의 (Individual Mission & Discussion)
각자 다른 방식으로 AI를 구현하고 결과를 비교
EXPERIMENT

각자 다른 도구나 알고리즘으로 AI를 구현하고, 이후 결과를 공유 및 비교하는 과정입니다. ChatGPT · Claude · Gemini 등 AI 도구는 팀의 새로운 멤버로 활용됩니다.

개인별 접근 방식 예시
🅰️
A 멤버: ChatGPT를 활용해 YOLO 기반 물체 인식 구현
🅱️
B 멤버: Claude로 코드 구조 설계 및 모듈 분리
🅲
C 멤버: Gemini로 멀티모달(이미지+텍스트) 접근
📊
비교 분석: 알고리즘별 집기 성공률 / 경로 탐색 효율 비교
👉 결과: 다양한 접근 방식 학습 + 비교 분석 능력
04
🤝
팀 퀴즈 (Team Quiz)
팀 단위로 협업하며 문제 원인을 분석
DEBUG

팀 단위로 문제를 해결하며, 협업을 통해 정답을 도출하는 활동입니다. 단순 암기가 아니라 "왜 실패했는가"를 같이 추론하는 디버깅 훈련입니다.

팀 퀴즈 예시
❓
로봇팔이 물체를 놓치는 이유는?
→ 센서 노이즈 / 좌표 오차 / 모델 오인식 중 어디가 원인일까?
❓
자율주행 로봇이 벽에 부딪히는 원인은?
→ 매핑 오류 / 센서 사각지대 / 제어 지연?
❓
RealSense Depth 값이 틀어지는 이유는?
→ 조명 / 반사면 / 캘리브레이션 문제?
👉 결과: 문제 분석 능력 + 협업 강화
05
✏️
개인 퀴즈 (Individual Quiz)
각자의 이해도를 점검하고 핵심 개념을 빠르게 확인
CHECK

각자의 이해도를 확인하고, 핵심 개념을 빠르게 점검하는 활동입니다. 짧고 명확하게, 매 차시 끝에 자연스럽게 진행됩니다.

개인 퀴즈 예시
📌
YOLO는 무엇을 하는 모델인가?
📌
Depth 카메라는 왜 필요한가?
📌
AI 모델이 현장에서 실패하는 가장 큰 이유는?
👉 결과: 개인 이해도 점검

Scenario Evolution
